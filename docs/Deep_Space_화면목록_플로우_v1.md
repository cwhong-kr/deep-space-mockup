# Deep Space 화면 목록 & 플로우 정의서

> 버전: v0.1 · 2026.04.22
> 기준: 1차 오픈 MVP 시나리오 (소셜로그인 ~ 출차)
> 총 화면 수: 32개 (메인 24개 + 모달/바텀시트 8개)

---

## 화면 넘버링 규칙

- **A**: 인증/온보딩
- **B**: 홈/대시보드
- **C**: 지점 검색
- **D**: 예약
- **E**: 길 안내
- **F**: 체크인/이용
- **G**: 퇴실/정산
- **H**: 마이페이지
- **M**: 모달/바텀시트 (공통)

---

## A. 인증 / 온보딩 (5개 화면)

### A-01. 스플래시

| 항목 | 내용 |
|------|------|
| 목적 | 브랜드 인지, 자동 로그인 체크 |
| 구성 요소 | DEEP 로고, "Time, Designed." 태그라인, 로딩 인디케이터 |
| 동작 | 토큰 있음 → B-01 홈 / 토큰 없음 → A-02 로그인 |
| 체류 시간 | 최대 2초 |

### A-02. 로그인

| 항목 | 내용 |
|------|------|
| 목적 | 소셜 로그인 진입 |
| 구성 요소 | DEEP 로고 + 타이틀 ("시간을 디자인하는 첫 걸음"), 카카오 로그인 버튼, Google 로그인 버튼, Apple 로그인 버튼, 이메일/휴대폰 로그인 텍스트 링크, 회원가입 텍스트 링크 |
| 플로우 | 소셜 버튼 탭 → 해당 OAuth 인증 → 기존 회원이면 B-01 / 신규면 A-03 |

### A-03. 회원가입 — 본인인증

| 항목 | 내용 |
|------|------|
| 목적 | 본인 확인 + 기본 정보 수집 |
| 구성 요소 | 진행 인디케이터 (1/3), 이름 (소셜에서 자동 입력), 휴대폰번호 입력 + 인증번호 발송 버튼, 인증번호 입력 필드 + 타이머 (3분), 서비스 이용약관 / 개인정보처리방침 동의 체크, 다음 버튼 |
| 플로우 | 인증 완료 + 동의 → A-04 |

### A-04. 회원가입 — 차량 등록

| 항목 | 내용 |
|------|------|
| 목적 | 차량 정보 등록 (충전 예상·LPR 체크인용) |
| 구성 요소 | 진행 인디케이터 (2/3), 차량번호 입력 (예: 12가 3456), 차종 선택 (제조사 → 모델 → 연식 드롭다운), 배터리 용량 자동 입력 (차종 선택 시), "나중에 등록하기" 스킵 링크, 다음 버튼 |
| 플로우 | 등록 완료 또는 스킵 → A-05 |
| 참고 | 차량 미등록 시 충전 잔량 예측·LPR 체크인 불가, 이후 마이페이지에서 등록 유도 |

### A-05. 회원가입 — 완료 & 알림 설정

| 항목 | 내용 |
|------|------|
| 목적 | 가입 완료 안내 + 푸시 권한 요청 |
| 구성 요소 | 진행 인디케이터 (3/3), 환영 메시지 ("환영합니다! 이제 충전하면서 일하는 시간이 시작됩니다"), 푸시 알림 수신 동의 (OS 권한 팝업 트리거), 위치 서비스 권한 요청 (지오펜스용), 시작하기 버튼 |
| 플로우 | → B-01 홈 |

## B. 홈 / 대시보드 (2개 화면)

### B-01. 홈 (예약 없음 상태)

| 항목 | 내용 |
|------|------|
| 목적 | 서비스 진입점, 현재 상태 요약 |
| 구성 요소 | 상단: 햄버거 메뉴 + DEEP 로고 + 알림 벨 아이콘 (읽지 않은 알림 뱃지), 차량 상태 카드: 차량번호·배터리 잔량 (차량 API 연동 시), 퀵 액션 그리드: 충전 / 라운지 / 길 안내 / 예약하기, 근처 지점 추천 카드 (거리 + 잔여 좌석), 공지사항 배너 (최신 1건), 하단 탭바: 홈·이용현황·예약·마이페이지 |
| 플로우 | 충전/라운지/예약하기 → C-01 / 알림 벨 → B-02 / 지점 카드 탭 → C-02 |

### B-02. 알림함

| 항목 | 내용 |
|------|------|
| 목적 | 수신 알림 목록 확인 |
| 구성 요소 | 알림 리스트 (최신순), 각 알림: 아이콘 + 제목 + 내용 미리보기 + 시간 + 읽음/안읽음 표시, 알림 유형: 예약확정·체크인·충전완료·종료임박·정산완료·공지, 알림 탭 시 해당 화면으로 이동 |
| 플로우 | 예약 알림 탭 → D-04 / 충전 알림 탭 → F-03 / 정산 알림 탭 → G-02 |

## C. 지점 검색 (3개 화면)

### C-01. 지점 검색 — 지도 뷰

| 항목 | 내용 |
|------|------|
| 목적 | 현위치 기반 DS 지점 탐색 |
| 구성 요소 | 상단 검색바 ("지점 또는 목적지를 검색하세요"), 지도 영역: 현위치 마커 + DS 지점 핀들 (잔여 좌석 수 표시), 지점 핀 색상: 초록(여유) / 주황(혼잡) / 빨강(만석), 하단 스와이프 패널: 지점 리스트 (카드형), 정렬 토글: 거리순 / 잔여 좌석순 / 충전기순, 현위치 재설정 버튼 (FAB) |
| 플로우 | 지점 핀 또는 카드 탭 → C-02 / 검색바 탭 → C-03 |

### C-02. 지점 상세

| 항목 | 내용 |
|------|------|
| 목적 | 선택 지점의 상세 정보 확인 |
| 구성 요소 | 지점 대표 이미지 (캐러셀), 지점명 + 주소 + 거리, 운영시간 (오늘 기준 강조), 실시간 현황 카드들: [라운지] 잔여 N석 / 전체 N석 + 좌석 배치도 썸네일, [회의실] 4인실·8인실 각각 예약 가능 여부, [충전기] 완속 7kW × N기 (잔여 N기) + 급속 50kW 유무, [편의시설] 아이콘 나열 (Wi-Fi·프린터·음료·사물함 등), 주차 요금 안내, 위치 약도 (주차장 내 DS 위치 — 층·구역), 리뷰/평점 (향후 확장), 하단 고정: "예약하기" 버튼 + "길 안내" 버튼 |
| 플로우 | 예약하기 → D-01 / 길 안내 → E-01 / 좌석 배치도 탭 → M-01 |

### C-03. 지점 검색 — 텍스트 검색

| 항목 | 내용 |
|------|------|
| 목적 | 키워드 기반 지점 검색 |
| 구성 요소 | 검색 입력 필드 (자동 포커스, 키보드 노출), 최근 검색어 (최대 5개 + 전체 삭제), 검색 결과 리스트 (지점명 하이라이트), 결과 없음 상태 ("검색 결과가 없습니다") |
| 플로우 | 검색 결과 탭 → C-02 |

## D. 예약 (5개 화면)

### D-01. 예약 — 날짜/시간 선택

| 항목 | 내용 |
|------|------|
| 목적 | 이용 날짜와 시간대 선택 |
| 구성 요소 | 선택된 지점명 (상단 고정), 캘린더 (월간 뷰, 오늘 이후만 선택 가능), 시작 시간 선택 (30분 단위 타임 슬롯), 종료 시간 선택 (최소 1시간 ~ 최대 4시간), 선택 시간대의 잔여 좌석/충전기 수 실시간 표시, 다음 버튼 |
| 플로우 | → D-02 |
| 예외 | 선택 시간대 만석 시: "해당 시간대는 예약이 어렵습니다" + 대체 시간 추천 |

### D-02. 예약 — 공간/충전기 선택

| 항목 | 내용 |
|------|------|
| 목적 | 좌석·회의실·충전기 선택 |
| 구성 요소 | 라운지 좌석 선택: 좌석 배치도 (선택 가능/불가 표시) 또는 자동 배정, 회의실 선택 (선택사항): 4인실·8인실 카드 (가격 표시), 충전기 선택: 충전기 번호 리스트 (잔여 기기만 활성), 선택 요약 하단 바: "좌석 A-7 · 충전기 C-03", 다음 버튼 |
| 플로우 | → D-03 |

### D-03. 예약 — 결제

| 항목 | 내용 |
|------|------|
| 목적 | 예약 확정 및 결제 |
| 구성 요소 | 예약 요약: 지점·날짜·시간·좌석·충전기, 예상 금액 내역: 라운지 N,000원 / 회의실 N,000원 / 충전 실비 별도 정산, 결제 수단 선택: 신용카드·카카오페이·네이버페이·토스 등 (PG 연동), 쿠폰/할인코드 입력 (2차 오픈 대비 UI만 구성, 비활성), 결제하기 버튼 |
| 플로우 | 결제 성공 → D-04 / 결제 실패 → M-02 |

### D-04. 예약 완료

| 항목 | 내용 |
|------|------|
| 목적 | 예약 확정 안내 |
| 구성 요소 | 완료 아이콘 (체크 애니메이션), "예약이 완료되었습니다", 예약 정보 카드: 예약번호·지점·날짜·시간·좌석·충전기, 충전 예상 안내: "현재 배터리 42% → 퇴실 시 약 85% 예상", 취소 안내: "이용 시작 30분 전까지 무료 취소 가능", 액션 버튼: "길 안내" (primary) / "홈으로" (secondary), 캘린더 추가 (iOS/Google 캘린더 연동) |
| 플로우 | 길 안내 → E-01 / 홈 → B-01 |
| Push | 예약 확정 Push 발송 (시나리오 문서 STEP 03 참고) |

### D-05. 예약 상세 / 관리

| 항목 | 내용 |
|------|------|
| 목적 | 기존 예약 조회·수정·취소 |
| 구성 요소 | 예약 정보 전체 (D-04와 동일), 예약 상태 뱃지: 확정·이용중·완료·취소, 길 안내 버튼, DS 위치 약도 (주차장 내), 체크인 버튼 (이용 시작 시간 전후 30분 내 활성), 예약 취소 버튼 (무료 취소 가능 시간 표시), 문의하기 링크 |
| 플로우 | 길 안내 → E-01 / 체크인 → F-01 / 취소 → M-03 |

## E. 길 안내 (1개 화면 + 1개 바텀시트)

### E-01. 길 안내 화면

| 항목 | 내용 |
|------|------|
| 목적 | 목적지 확인 + 네비 연동 + 도착 잔량 안내 |
| 구성 요소 | 목적지 요약: 지점명 + 주소, 지도 미리보기 (현위치 → 목적지 경로 표시), 예상 소요 시간 + 거리, 도착 예상 배터리: "현재 42% → 도착 시 약 38% (예상 소모 4%)", 충전 안내: "도착 후 바로 충전하면 퇴실 시 약 85%", "내비로 안내 시작" 버튼 |
| 플로우 | 내비 버튼 → M-04 (네비 선택 바텀시트) |
| 참고 | 배터리 잔량은 차량 API 연동 시 실시간, 미연동 시 마지막 수동 입력값 사용 |

### M-04. 네비게이션 선택 (바텀시트)

| 항목 | 내용 |
|------|------|
| 구성 요소 | 타이틀: "어떤 내비로 안내할까요?", 티맵 (설치 여부 표시), 카카오내비 (설치 여부 표시), 네이버 지도 (설치 여부 표시), 차량 내비게이션 (차종에 따라 분기), 미설치 앱: 비활성 + "설치 필요" 표시 |
| 동작 | 선택 시 해당 앱 딥링크로 전환 (목적지 좌표 전달) |

## F. 체크인 / 이용 (4개 화면)

### F-01. 체크인 화면

| 항목 | 내용 |
|------|------|
| 목적 | DS 출입문 체크인 |
| 구성 요소 | 탭 전환: QR 코드 / 출입번호, [QR 탭] QR 코드 이미지 (실시간 생성, 1분마다 갱신), QR 사용 안내: "DS 출입문 리더기에 스캔해주세요", [출입번호 탭] 4자리 숫자 크게 표시 (예: 7 2 9 4), 출입번호 안내: "출입문 키패드에 입력해주세요", 유효시간 타이머 (5분), 예약 정보 요약 (하단) |
| 플로우 | 체크인 성공 (서버 확인) → F-02 |
| 트리거 | 지오펜스 Push 클릭 → D-05 → 체크인 버튼 → F-01 |

### F-02. 체크인 완료

| 항목 | 내용 |
|------|------|
| 목적 | 체크인 확인 + 이용 시작 안내 |
| 구성 요소 | 완료 아이콘 + "환영합니다! 시간을 디자인하세요", 체크인 시간, 이용 정보: 좌석 번호·충전기 번호·이용 종료 시간, Wi-Fi 정보 카드: SSID + 비밀번호 (복사 버튼), 충전 시작 안내: "충전이 시작되었습니다", "이용 현황 보기" 버튼 |
| 플로우 | → F-03 |
| Push | 체크인 완료 Push 발송 (시나리오 문서 STEP 06 참고) |

### F-03. 이용 현황 (이용 중 메인)

| 항목 | 내용 |
|------|------|
| 목적 | 이용 중 실시간 상태 확인 + 부가 서비스 |
| 구성 요소 | 충전 상태 카드: 프로그레스 원형/바 + 현재 % + 예상 완료 시간 + 충전 속도 (kW), 공간 이용 카드: 좌석 번호 + 잔여 시간 카운트다운 + 시간 연장 버튼, 퀵 액션: 시간 연장 (+30분/+1시간) / 음료 주문 (2차 오픈) / 회의실 추가, Wi-Fi 정보 (접기/펼치기), 이용 종료 버튼 (하단), 편의시설 안내 (프린터 위치 등) |
| 플로우 | 시간 연장 → M-05 / 이용 종료 → G-01 |
| Push | 종료 10분 전 Push (시나리오 문서 STEP 08 참고) |

### F-04. 충전 상세

| 항목 | 내용 |
|------|------|
| 목적 | 충전 진행 상세 정보 |
| 구성 요소 | 충전 프로그레스 (큰 원형 게이지), 시작 배터리 % → 현재 % → 목표 %, 충전 시작 시간 / 예상 완료 시간, 누적 충전량 (kWh), 충전 속도 (kW), 예상 충전 비용, 충전 중지 버튼 (긴급 시) |
